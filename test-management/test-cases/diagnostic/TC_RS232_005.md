## Description

This test validates RS-232 communication between the POS terminal and connected peripherals (scanners, printers, scales, PIN pads). It verifies that the serial communication parameters (baud rate, data bits, stop bits, parity, protocol) are correctly configured and that data is transmitted and received properly.

**Reference:** RS-232 Configuration Guide - diags2x20.properties Template

---

## Prerequisites

### Hardware Requirements
- [ ] Toshiba POS terminal (TCx 810, TCx 700, TCx 300, or SurePOS series)
- [ ] RS-232 peripheral (one of: scanner, printer, scale, PIN pad)
- [ ] Serial cable connecting the peripheral to the POS terminal
- [ ] Toshiba Diagnostics USB key with Minimal UI configuration

### Software Requirements
- [ ] Toshiba Diagnostics Minimal UI bootable USB key
- [ ] `diags2x20.properties` file with correct RS-232 parameters
- [ ] Peripheral-specific diagnostic tools (if applicable)

### Environment
- [ ] POS terminal powered on and connected to power
- [ ] Peripheral powered on and connected via RS-232
- [ ] 2x20 display connected and operational

## Configuration Parameters

This test requires the following `diags2x20.properties` parameters:

```properties
# RS-232 Communication Parameters
baudrate=9600
databits=8
stopbits=1
parity=None
protocol=XON/XOFF

# Peripheral Configuration
printer.model=TCx80
scanner.model=Honeywell
scale.model=Micros

# Diagnostic Settings
diagnostic.timeout=30
diagnostic.retries=3

Note: Adjust parameters based on the specific peripheral being tested.

Test Steps (Given/When/Then)
GIVEN
The Toshiba Diagnostics USB key is inserted into the POS terminal

The RS-232 peripheral is connected to the POS terminal via serial cable

The peripheral is powered on and ready

The diags2x20.properties file is configured with correct RS-232 parameters for the peripheral

The terminal is powered off

WHEN
Power on the POS terminal

The terminal boots from the USB key

Minimal UI displays on the 2x20 screen

Authenticate (if required)

Complete iButton authentication if enabled

Navigate to the RS-232 Test Menu

Use navigation keys to find the RS-232/peripheral test section

Select the appropriate peripheral type (Scanner, Printer, Scale)

Send a Test Command

For Scanner: Scan a test barcode

For Printer: Send a test print command

For Scale: Request weight reading

For PIN Pad: Send a test command (e.g., display message)

Verify the Response

Check the 2x20 display for the received data

For Printer: Verify the printed output

For Scanner: Verify the scanned data is displayed

For Scale: Verify the weight reading

THEN
Successful Communication Scenario

The peripheral responds within the diagnostic timeout period

The data received matches the expected format

No error messages are displayed on the 2x20 screen

For Printer: The test receipt prints correctly (proper formatting, no garbled text)

For Scanner: The scanned barcode data is correctly displayed

For Scale: The weight reading is accurate and in the correct format

Communication Failure Scenario

The 2x20 display shows a timeout or communication error

No data is received from the peripheral

An error code is displayed (e.g., "ERR-001: Communication timeout")

The diagnostic log contains relevant error information

Invalid Configuration Scenario

Garbled or incorrect data is received

Peripheral does not respond (timeout)

Error message indicates configuration mismatch

Troubleshooting Guide
Symptom: No Response from Peripheral
Possible Cause	Solution
Baud rate mismatch	Verify baud rate matches peripheral configuration
Incorrect cable	Check cable type (null modem vs. straight-through)
Peripheral not powered	Verify power to the peripheral
Wrong COM port	Check serial port assignment in diagnostics
Symptom: Garbled Data
Possible Cause	Solution
Data bits mismatch	Verify databits (should be 7 or 8)
Parity mismatch	Check parity setting (None, Even, Odd)
Stop bits mismatch	Verify stopbits (1 or 2)
Flow control mismatch	Try XON/XOFF vs DTR/DSR
Symptom: Timeout Error
Possible Cause	Solution
Cable not connected	Check physical connection
Peripheral not responding	Test peripheral on another system
Incorrect protocol	Verify protocol setting
Diagnostic timeout too short	Increase diagnostic.timeout value
